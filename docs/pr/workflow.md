# PR Workflow

## 목적
QA를 통과한 변경사항만 PR로 생성하고,
PR 생성 이후 리뷰 및 재검토 루프까지 일관되게 관리한다.

## 규칙
- QA 통과 전에는 PR을 만들지 않는다
- PR 제목과 본문은 한국어로 작성한다
- PR에는 변경 내용 / 변경 이유 / 확인 포인트를 포함한다
- 리뷰어가 확인해야 할 위험 요소를 함께 적는다
- PR 생성 후 Codex 코드 리뷰를 활용할 수 있다
- 리뷰 반영 후 필요하면 재검토를 요청한다

## Agent Rules

PR 에이전트는:
1. 현재 브랜치 상태를 확인한다
2. QA 결과를 확인한다
3. 브랜치명, 커밋 단위, PR 제목/본문을 정리한다
4. 사용자 승인 후 push 및 PR 생성을 진행한다

## Review Loop

PR 생성 후:
1. 자동 코드 리뷰가 활성화되어 있으면 Codex 리뷰를 기다린다
2. 필요 시 PR 댓글로 `@codex review`를 사용해 수동 리뷰를 요청한다
3. 리뷰 결과를 바탕으로 로컬 멀티에이전트가 수정 작업을 진행한다
4. 수정 후 QA를 다시 수행한다
5. 같은 PR에 변경사항을 push 한다
6. 필요 시 다시 `@codex review`로 재검토를 요청한다

## Manual Review Request
